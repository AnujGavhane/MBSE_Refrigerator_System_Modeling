# Refrigerator System Modeling using MBSE and SysML

## Overview

This project demonstrates the application of Model-Based Systems Engineering (MBSE) principles to a Refrigerator System using SysML and Cameo Systems Modeler.

The refrigerator is modeled as a complete engineering system with defined requirements, stakeholders, use cases, system context, and physical architecture.

## Project Objectives

- Apply MBSE methodology to a real-world system.
- Model refrigerator requirements using SysML.
- Develop system architecture and traceability.
- Create Use Case, Requirement, and Package Diagrams.
- Analyze the physical hierarchy of refrigerator components.

## Tools Used

- Cameo Systems Modeler
- SysML
- UML
- Model-Based Systems Engineering (MBSE)

## System Description

The refrigerator system maintains food and beverage storage at safe temperatures through a refrigeration cycle. 
The system monitors temperature, manages cooling operations, and interacts with users through control interfaces.

## Project Deliverables

### 1. Requirement Diagram
Captures functional and non-functional requirements of the refrigerator system.

### 2. Requirement Table
Provides requirement traceability and documentation.

### 3. Use Case Diagram
Models interactions between users and the refrigerator system.

### 4. Package Diagram
Organizes system elements into logical packages.

### 5. System Context Diagram
Represents external entities interacting with the refrigerator.

### 6. Physical Hierarchy Diagram
Shows critical hardware components and subsystem relationships.
